## 提交问题

### 1. 问题报告模板

**使用GitHub Issues**:
```markdown
## 问题描述
简要描述遇到的问题

## 环境信息
- 操作系统: [e.g., Ubuntu 20.04]
- PHP版本: [e.g., 8.1.15]
- 数据库版本: [e.g., MySQL 8.0]
- 应用版本: [e.g., v2.1.0]

## 复现步骤
1. 访问 [URL/执行操作]
2. 点击 [按钮/链接]
3. 滚动到 [位置]
4. 看到错误

## 期望行为
描述您期望发生的情况

## 实际行为
描述实际发生的情况

## 错误日志
```
粘贴相关错误日志
```

## 截图
如果适用，请提供截图

## 其他信息
添加任何其他有用的信息
```

**使用邮件支持**:
主题格式: `[类型] 简要问题描述`

类型标识:
- `[BUG]` - 错误报告
- `[FEATURE]` - 功能请求
- `[QUESTION]` - 使用问题
- `[URGENT]` - 紧急问题

### 2. 问题分类

**按严重程度分类**:

**P0 - 紧急** (系统完全不可用):
- 系统无法启动
- 数据丢失风险
- 安全漏洞
- 支付系统故障

**P1 - 高优先级** (核心功能受影响):
- 主要功能无法使用
- 性能严重下降
- 大量用户受影响

**P2 - 中优先级** (部分功能受影响):
- 次要功能异常
- 特定场景下的问题
- 少量用户受影响

**P3 - 低优先级** (一般问题):
- 使用咨询
- 功能建议
- 文档问题

### 3. 提交前检查清单

**在提交问题前，请确认**:

- [ ] 已搜索现有问题和文档
- [ ] 复现步骤清晰可执行
- [ ] 提供了完整的错误信息
- [ ] 包含了环境版本信息
- [ ] 附上了相关日志和截图
- [ ] 使用了合适的分类和标签

**常见问题检查**:
```bash
# 检查系统状态
php artisan --version
php -v
composer --version

# 检查配置
php artisan config:show
php artisan route:list

# 检查日志
tail -n 50 storage/logs/laravel.log
tail -n 50 /var/log/nginx/error.log

# 检查数据库连接
php artisan tinker
>>> DB::connection()->getPdo();
```

## 问题跟踪

### 1. 状态定义

**问题状态**:
- **New** - 新提交的问题
- **Triage** - 正在评估和分类
- **Confirmed** - 已确认的问题
- **In Progress** - 正在处理中
- **Review** - 等待代码审查
- **Testing** - 等待测试验证
- **Resolved** - 问题已解决
- **Closed** - 问题已关闭

**处理流程**:
```
New → Triage → Confirmed → In Progress → Review → Testing → Resolved → Closed
  ↓
Triage → Won't Fix (无效问题)
Triage → Duplicate (重复问题)
In Progress → Blocked (被阻塞)
```

### 2. 优先级处理

**优先级规则**:
1. P0问题立即处理
2. P1问题24小时内处理
3. P2问题72小时内处理
4. P3问题按计划处理

**升级条件**:
- 问题影响扩大
- 长时间未解决
- 客户强烈要求
- 业务影响严重

### 3. 沟通更新

**更新频率**:
- P0: 每4小时更新
- P1: 每天更新
- P2: 每2天更新
- P3: 每周更新

**更新内容**:
- 当前处理状态
- 已完成的步骤
- 遇到的问题
- 下一步计划
- 预计解决时间
